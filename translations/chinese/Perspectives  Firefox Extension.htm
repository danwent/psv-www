Perspectives : Mozilla 火狐插件

概述

我们为流行的火狐浏览器开发了一个插件。该插件会在您连接HTTPS站点时联系认证服务器。

对于Perspectives怎样工作，您可以浏览我们的 Perspectives主页。

该插件主要提供两项功能：

  1. 如果您连接一个不可信的主页（例如使用自分配的认证），火狐将会给您一个十分两人讨厌的安全错误，并强迫您人工的安装一个“异常”。Perspectives能够检测自分配的认真是否可用，并且如果足够安全的话，Perspectives能帮助自动的跳过这些安全错误页面。
  2. 完全有可能一个黑客欺骗火狐信任的许多认证机构中的一个，并使之对一个可以信任的主页使用不正确的认证。在这种情况下，Perspectives能够检测攻击。并且如果发现任何可疑的事件，Perspectives会给您警告。

* 如果HTTPS站点认证过期，或者域名不匹配（例如www.gmail.com使用了mail.google.com的认证），同样适用。

使用方法

Perspectives在您浏览器窗口的右下角提供了一个小的信息图标。通过该图标您可以看到认证客户端插件的工作。该图标会告诉您该认证是否被确认，并且通过该图标您能够查看对于每一个服务器反馈的详细信息。双击这个信息图标，您可以得到详细的认证信息，并能打开配置菜单。

默认情况，Perspectives仅仅对“自分配认证”和火狐认为不可用的认证询问认证服务器。您能改变配置选项，对于所有HTTPS链接询问认证服务器。配置选项还能够让您控制安全级别，该安全级别决定由Perspectives使用的确认认证是否可用的标准。

Perspectives会在本地保存“可信”认证的认证数据。您可以配置Perspectives，使Perspectives永久保存这些数据或者只当前保存（默认设置）。

Perspectives能够通过自动更新来修复软件中的错误，更新特性和认证配置数据。如果您在使用中遇到问题，请确认您使用最新版本（通过“工具”，“附加组件”，“查找更新”进行更新），并在更新可用时安装更新。

请看Perspectives火狐帮助页面。

使用需知：

  1. 如果您的局域网使用代理或者防火墙来连接因特网，代理或者防火墙有可能阻止Perspectives连接认证服务器，从而导致所有的主页认证都失败(详细信息)。
  2. 当询问认证服务器时，如果您遇到下面的信息："Perspectives component (@cs.cmu.edu/Perspectives;1) not installed correctly. Please report information about your platform to developers", pleases visit 请访问该页面。
  3. Gentoo用户报告说当使用由“PORTAGE”下载的火狐浏览器运行perspectives时会遇到问题。我们还没有解决方案，但是如果用户使用从Mozilla下载的火狐二进制文件安装的火狐浏览器的话，perspectives能够在Gentoo上正常运行。
  4. 有用户报告Perspectives会和Foxit Reader 用于火狐的PDF浏览器插件互相干扰。原因我们还不知道，但是卸载Foxit Reader插件后，Perspectives能正常工作。
  5. 火狐2和火狐3所使用的插件API有很大的差异，现在的代码只在火狐3上工作，火狐2不支持。
  6. 我们用于跳过火狐安全警告页面（由自分配认证、不匹配以及过期的认证引起）的代码并不完善，有时您会看待警告在我们的代码跳过它之前出现。

源代码

火狐插件源代码

Perspectives火狐插件包含C++ XPCOM组件，因此它必须在目标平台上编译。Windows需要Visual C++ 2008 Express Edition，免费下载。

现在，有关编译的文档并不完善。但是您在编译“firefox-v2/xp_com”目录代码之前，必须编译“common”和“notary-client”目录内的代码。Windows下的代码是在“firefox-v2/win_xp_com”目录，但是如果两套代码合并在一起，这个目录将改变。
